# 目的
情報を見た相手が、短時間で要点を理解し、誤解なく判断できる状態をつくる。

このスキルは、見た目を飾ることではなく、
「伝わること」「比較しやすいこと」「判断しやすいこと」を優先する。

# 適用場面
- PowerPointの構成案作成
- レポート本文の見出し、図表、表現の改善
- ダッシュボードのページ設計
- グラフ種別の選定
- KPI資料のレビュー
- 提案書の図表や要約パートの改善
- 既存資料のリライトや情報整理

# 基本姿勢
- 装飾より理解の速さを優先する
- 情報量より判断しやすさを優先する
- きれいさだけでなく、誤解の少なさを重視する
- 作り手の都合ではなく、読み手の視点で構成する
- 1ページ、1スライド、1セクションごとに「何を伝えるか」を明確にする

# 優先原則
## 1. 主メッセージを先に決める
- 各ページやスライドには、主メッセージを1つだけ置く
- まず「このページは何を伝えるページか」を1文で言える状態にする
- その1文で言えない内容は、情報が混ざっている可能性が高い

## 2. 比較しやすさを優先する
- 見栄えよりも比較しやすい構造を選ぶ
- 数字の意味、差分、傾向、順位、変化がすぐ見えるようにする
- 読み手が頭の中で補完しなくても済む構成にする

## 3. 情報を詰め込みすぎない
- 重要情報を埋もれさせない
- 1ページに複数の論点を入れすぎない
- 補足情報は主メッセージを邪魔しない位置に置く

## 4. 意味を言葉で補う
- 色や位置だけに意味を持たせない
- 数値には単位をつける
- グラフには対象、期間、粒度を明記する
- 読み手が前提知識なしでも理解できるようにする

# 構成ルール
## スライド・ページ構成
- 最初に結論か要点を示す
- その後に根拠、内訳、補足を置く
- 情報は「結論 → 理由 → 補足」の順に並べる
- 重要情報は左上または冒頭に置く
- 関連する要素は近くにまとめる
- 無関係な情報を同じ塊にしない

## レポート本文構成
- 見出しだけで流れがわかるようにする
- 1段落1メッセージを基本とする
- 長い説明の前に、要点を短く置く
- 図表の前後で「何を見るべきか」を短く補う

# グラフ選定ルール
## 時系列の変化
- 推移を見せたいときは折れ線グラフを優先する
- 複数系列を重ねすぎない
- 比較対象が多い場合は分割や絞り込みを検討する

## 項目比較
- カテゴリ間の大小比較は棒グラフを優先する
- 項目名が長い場合は横棒グラフを優先する
- 順位比較では降順に並べる

## 構成比
- 構成比は円グラフより棒グラフや積み上げ棒を優先する
- 円グラフは2〜3項目程度で直感性が重要な場合のみ許容する
- 構成比だけでなく、必要に応じて実数も併記する

## 差分・増減
- 前週比、前年比、計画差などは差分がすぐ見える形にする
- 増減率だけでなく、必要なら実数差も示す
- 良い増減か悪い増減かが文脈でわかるようにする

## 棒グラフの扱い
- 棒グラフの軸は原則0始点とする
- 例外を使う場合は誤解を招かないよう注記する
- 並び順に意味があるなら明示する

# タイトルルール
- 曖昧なタイトルを避ける
- タイトルだけで何の数字かわかるようにする
- 対象、期間、粒度、単位をできるだけ含める
- 比較のグラフでは比較基準を明示する
- 「結果」「実績」など抽象語だけで終わらない

## 悪い例
- 売上
- 実績
- 比較
- 申込状況

## 良い例
- 月次売上推移（税抜、2026年1月〜3月）
- 媒体別申込件数（今週実績）
- 前年同月比で見たCV数の増減
- 地域別資料請求件数ランキング

# 数字の見せ方ルール
- 数字には単位を必ずつける
- 百分率と実数を混同させない
- 桁区切りを適切に入れる
- 不要に細かい小数点を出しすぎない
- 指標の定義がブレる場合は注記する
- 比率だけで誤解が起きる場合は分母も補う

# 色ルール
- 強調色は絞る
- 1ページで意味のある色は必要最小限にする
- 赤は警告、未達、異常などに限定する
- 補助情報は抑えた色で扱う
- 色だけで意味を伝えず、ラベル、注釈、凡例を併用する
- ページごとに色の意味を変えない
- 目立たせたいからという理由だけで多色化しない

# レイアウトルール
- 視線の流れを邪魔しない
- 余白を確保する
- タイトル、本文、図表、注記の役割を分ける
- 整列を揃える
- 似た情報は似た見せ方に統一する
- 装飾要素は、理解を助ける場合のみ使う

# 表のルール
- 行と列の意味が一目でわかるようにする
- 桁数や単位を揃える
- 強調したいセルだけを限定して目立たせる
- 表でしか伝わらないときだけ表を使う
- 傾向を見る用途ならグラフ化を検討する

# 禁止事項
- 3Dグラフを使わない
- 円グラフを多用しない
- 単位なしの数値を置かない
- 装飾目的だけのアイコンや色を増やしすぎない
- 同じページに論点を詰め込みすぎない
- 長いタイトルで意味がぼやける書き方をしない
- 色だけで良し悪しを伝えようとしない
- 誤読を招く軸設定や並び順を使わない
- 読み手に前提知識を要求しすぎない

# 例外条件
- 経営層向けなどで一瞬の直感理解が重要な場合は、簡潔さを優先して情報を減らしてよい
- 構成比が2〜3項目で直感性重視なら円グラフを許容する
- 棒グラフの0始点が比較性を大きく下げる場合は例外的に調整可。ただし誤解を避ける注記を入れる
- デザインよりブランド整合性が強く求められる場合は、色運用を最小限調整してよい。ただし比較しやすさを損なわないこと

## 経営層向け資料の補足ルール
- 1スライド1判断をさらに徹底し、説明より意思決定に必要な情報を優先する
- 「全体像の網羅」より「行動が変わる論点」を残す
- 将来見込みを示すときは、予測と現状を混同させないタイトルにする
- 経営層向けでは、タイトルも単なる項目名ではなく判断が伝わる文に寄せる
- 現状報告と将来見通しを混同させないため、見込みを示すときは根拠となる現状指標を明示する

# レビュー観点
以下の順で確認する。

## 1. 主メッセージ
- このページやスライドの主メッセージは1文で言えるか
- その主メッセージはタイトルや見出しに反映されているか

## 2. 情報量
- 情報が多すぎないか
- 重要情報が埋もれていないか
- 不要な補足が主役を邪魔していないか

## 3. 図表選定
- グラフ種別は目的に合っているか
- 比較、推移、構成比、差分の見せ方が適切か
- 表のままにすべきか、グラフ化すべきか判断できているか

## 4. タイトルと注記
- 対象、期間、粒度、単位が入っているか
- 比較基準が明確か
- 誤解を招く余地がないか

## 5. 色とレイアウト
- 強調色が増えすぎていないか
- 色の意味が一貫しているか
- 視線の流れが自然か
- 余白や整列が崩れていないか

# 出力時の指示
このスキルを使って提案やレビューを行うときは、以下を優先する。
- まず主メッセージを定義する
- 次に構成と図表種別を決める
- その後にタイトル、色、レイアウトを整える
- 必要なら最後に禁止事項とチェック観点でレビューする
